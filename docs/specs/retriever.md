# Спецификация - Retrieval / Сборка контекста

## 1. Ответственность

В текущем PoC retrieval реализован через `ContextAssembler`. Его задача - собрать read-only снимок контекста из локальных CSV-таблиц и передать его агентам как системный контекст.

Это не vector retrieval: в системе нет embeddings, индекса и semantic search. Источники данных структурированы и пока небольшие, поэтому контекст собирается простыми правилами: чтение таблиц, фильтрация, сортировка и форматирование.

## 2. Источники данных

| Источник | Как используется |
|---|---|
| `fridge.csv` | Основной инвентарь, сортировка по сроку годности |
| `pantry.csv` | Краткий список продуктов в кладовой |
| `freezer.csv` | Краткий список продуктов в морозилке |
| `people.csv` | Цели, диеты и ограничения членов семьи |
| `ingredients.csv` | Аллергии, dislikes и preferences |
| `dishes.csv` | Ротационные блюда и предпочтения |
| `history.csv` | Недавние приемы пищи для проверки повторов |
| `shopping_list.csv` | Контекст для ShoppingAgent |
| `settings.json` | Язык ответа и пользовательские настройки |

## 3. Правила отбора

- `fridge.csv` сортируется по `expiry_date`; продукты без даты идут ниже.
- Продукты, срок годности которых истекает в течение `EXPIRY_WARNING_DAYS=2`, помечаются как жесткое ограничение.
- `pantry.csv` и `freezer.csv` показываются компактным списком.
- Ингредиенты с `preference_level in ["dislike", "allergy"]` попадают в блок `DO NOT USE`.
- Ротационные блюда выбираются по `is_rotation in ["true", "1", "yes"]`.
- История недавней еды ограничена последними 7 записями с `action="eaten"`.
- Cold start определяется, когда `people.csv`, `fridge.csv` и `pantry.csv` пустые.

## 4. Контракт выхода

`ContextAssembler.get_context_snapshot() -> str`

Возвращаемая строка должна содержать секции:

- `EXPIRING SOON`;
- `INVENTORY`;
- `HOUSEHOLD`;
- `RESTRICTIONS`;
- `ROTATION DISHES`;
- `RECENT HISTORY`.

Контекст должен быть безопасен для передачи как `system` message и не должен менять состояние системы.

## 5. Ограничения

- Нет подсчета token budget.
- Нет semantic ranking.
- Нет поиска рецептов во внешних источниках.
- Нет timestamp свежести контекста.
- Нет разграничения доступа по пользователям.
- Большие CSV-файлы могут переполнить контекст модели.

## 6. Планируемое развитие

Ближайшее развитие retrieval-контуры связано не с vector search, а с внешними источниками:

- интеграция с магазинами для наличия, цен и покупок;
- интеграция с источниками пищевой ценности для калорий и БЖУ;
- более точный расчет nutrition log;
- учет цен и доступности продуктов при планировании меню.

Если объем локальных рецептов, истории и блюд существенно вырастет, можно будет добавить отдельный слой отбора контекста, но это не главный следующий шаг.

## 7. Guardrails

- Сборка контекста только читает данные.
- Аллергии и dislikes нельзя удалять из контекста раньше менее важных данных.
- Продукты с истекающим сроком годности должны оставаться видимыми для meal planning.
- Если в будущем появится ограничение по токенам, сначала нужно сокращать историю и рецепты, а не ограничения и рискованные продукты.
